About Movie Recommender

   This recommender engine was created using the "small" subset of the GroupLens MovieLens 25 Million dataset. This "small" dataset contains approximately 100,000 ratings and 4,000 user defined movie 'tags' of approximately 10,000 movies by approximately 600 users. The Movie Recommender is a private, educational website. It is not for any commercial purpose and is not endorsed by any organization. The datasets are provided free for non-commercial public use by GroupLens MovieLens. For more information about the dataset, please visit the Data page.


How It Works

   For this recommender engine, keywords consist of movie 'tags' and genres. The engine first checks if the user input movie has any keyword matches within the dataset. If it does, then the recommender uses a Feature Analysis algorithm (cosine similarity) to recommend movies with similar keywords (tags and genres). This keyword-based algorithm maps all of the keywords and turns them into vectors. It then compares all of these vectors by measuring the angle between each vector. Movie-vectors with the smallest angle between them are, ideally, the most similar and are then recommended to the user.

   If there are no keyword matches, then the recommender engine uses a rating-based algorithm using Nearest Neighbors from the Machine Learning Python Library Sci-Kit Learn (sklearn.neighbors). It is an unsupervised algorithm that maps all of the ratings and turns them into vectors. It then compares all of these vectors by measuring the angle between each vector and again recommending the movies with the most similar vectors.

  The keyword-based algorithm is fairly accurate - it at least provides movies you expect. However, due to the limited tag data, it only has a recommendable universe of about 1,600 movies. If your movie isn't in there, it's got nothing for you.

  The rating-based algorithm is less accurate and often provides a mix of recommendations that are intuitive and sometimes bizarre. This is because the rating algorithm is solely based on dataset user ratings of movies. It doesn't matter if those movies have anything in common with the one you searched for. If users in the dataset rated it, your recommendations are based on those user's ratings of other movies. This creates some interesting recommendations!

  Part of the reason for this is due to the relatively narrow dataset used. The ratings are from 600 users. The average number of ratings per movie is 10 ratings while the median number of ratings is 3 ratings.

  This means more than 50% of the movies in the data - over 5,000 movies - Movie Recommender uses to provide recommendations have 3 ratings or less. This is called data sparsity - where, despite having a large data set, it is highly concentrated in only a few areas or datapoints. Nevertheless, movies with more ratings are much more likely to be recommended, as they are found in more vectors mapped by the algorithm. In this way, Movie Recommender can still provide meaningful recommendations when the input movie has close vector matches to the movies with many ratings in the dataset.
